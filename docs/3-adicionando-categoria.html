🎯 Modulo 1: Objetivo da mudança

Contexto: agora o projeto ganhou uma nova tabela chamada Categoria.

Meta desta trilha: consolidar a criação da tabela no schema, aplicar no banco e validar que tudo foi gerado corretamente.

🆕 Nova tabela: Categoria 📦 Tabela existente: Produto 🧪 Foco: schema + migrate + generate

🗂️ Modulo 2: Schema completo atualizado

📘 Importante: abaixo está o schema completo atual com a nova tabela, no caminho completo do projeto.

generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "sqlite"
}

model Categoria {
  id           Int      @id @default(autoincrement())
  nome         String   @unique
  descricao    String
  criadoEm     DateTime @default(now())
  atualizadoEm DateTime @updatedAt
}

model Produto {
  id           Int      @id @default(autoincrement())
  nome         String   @unique
  quantidade   Int
  preco        Float
  criadoEm     DateTime @default(now())
  atualizadoEm DateTime @updatedAt
}

⚙️ Modulo 3: Aplicar a nova tabela no banco

1. Validar schema
npx prisma validate
2. Formatar schema
npx prisma format
3. Criar migration da tabela nova
npx prisma migrate dev --name adiciona_tabela_categoria
4. Garantir client atualizado
npx prisma generate

Checklist rapido
