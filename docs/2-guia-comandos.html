Fase 1: A Fundacao e a Escolha do Banco

1. Prisma CLI (dev)
npm install prisma --save-dev

Baixa o motor do Prisma (CLI) para desenvolvimento.

2. Prisma Client (runtime)
npm install @prisma/client

Biblioteca usada pelo Next.js para conversar com o banco.

3. Prisma init com provider
npx prisma init --datasource-provider sqlite npx prisma init --datasource-provider mysql npx prisma init --datasource-provider postgresql

Escolha apenas um comando conforme o banco desejado.

Fase 2: O Desenho das Tabelas

Edite o `schema.prisma` com seus modelos e rode:

4. Formatar schema
npx prisma format

Alinha e organiza o arquivo do schema.

5. Validar schema
npx prisma validate

Detecta erros de tipo, sintaxe e relacionamentos antes de migrar.

Fase 3: Sincronizacao (Criacao Fisica)

6. Push sem historico
npx prisma db push

Empurra o schema direto para o banco, sem migration files.

7. Migration de desenvolvimento
npx prisma migrate dev --name init

Cria SQL versionado do que mudou no banco.

8. Gerar client
npx prisma generate

Gera tipos TypeScript/autocomplete do Prisma Client.

Fase 4: Gerenciamento e Testes Iniciais

9. Prisma Studio
npx prisma studio

Abre painel web para visualizar e editar registros.

10. Seed do banco
npx prisma db seed

Executa script de dados iniciais (ex: `seed.ts`).

Fase 5: Manutencao e Troubleshooting

11. Reset local
npx prisma migrate reset

Recria banco local com base nas migrations + seed.

12. Engenharia reversa
npx prisma db pull

Lê banco existente e atualiza `schema.prisma`.

13. Resolver migration quebrada
npx prisma migrate resolve

Marca status de migration quando ocorre falha no meio do processo.

Fase 6: Deploy (Producao)

14. Aplicar migrations em producao
npx prisma migrate deploy

Comando seguro para aplicar migrations pendentes no banco real.

Resultado esperado

Você domina o ciclo completo do Prisma: setup, modelagem, migração, testes, recuperação e deploy.
